IINTS-AF SDK

End-User Guide

Complete, practical documentation for researchers, engineers, and clinical users.

What this SDK is

IINTS-AF is a safety-first simulation and validation platform for insulin dosing algorithms. It provides deterministic safety rails, auditing, and clinical report generation before anything touches a real patient.

Core capabilities

  • Clinic-safe presets and patient profiles
  • Supervisor safety guardrails with reasons
  • Audit trail export (CSV/JSONL)
  • Baseline comparison vs. PID/standard pump
  • PDF clinical reports with branded visuals

Who it is for

  • Clinical researchers validating new strategies
  • ML engineers benchmarking AI controllers
  • Developers building decision-support tooling

Installation

python -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
pip install iints-sdk-python35

Optional Torch support: pip install iints-sdk-python35[torch]

From source (editable): pip install -e .

Quickstart (CLI)

iints quickstart --project-name iints_quickstart
cd iints_quickstart
iints presets run --name baseline_t1d --algo algorithms/example_algorithm.py

Outputs are saved to results/ inside the project.

iints run-full --algo algorithms/example_algorithm.py --duration 240 --output-dir results/run_full

Quickstart (Python)

import iints
from iints.core.algorithms.pid_controller import PIDController

outputs = iints.run_full(
    algorithm=PIDController(),
    scenario="scenarios/example_scenario.json",
    patient_config="default_patient",
    duration_minutes=240,
    seed=42,
    output_dir="results/quick_run",
)
print(outputs["report_pdf"])

Python API Reference

  • iints.run_simulation(...) returns results + audit + report when output_dir is set.
  • iints.run_full(...) always exports results + audit + baseline + PDF.
  • iints.Simulator(...) for custom loops and live stepping.
  • Simulator.save_state() and Simulator.load_state() for time-travel debugging.
  • iints.generate_report(...) for PDF only.

Algorithm Interface

Implement predict_insulin and return a dict with total_insulin_delivered. 

from iints.api.base_algorithm import InsulinAlgorithm, AlgorithmInput

class MyAlgorithm(InsulinAlgorithm):
    def predict_insulin(self, data: AlgorithmInput):
        if data.current_glucose > 180:
            return {"total_insulin_delivered": 1.0, "bolus_insulin": 1.0}
        return {"total_insulin_delivered": 0.0}

    {"start_time": 30, "event_type": "meal", "value": 75, "absorption_delay_minutes": 180}

Valid event types: meal, missed_meal, sensor_error, exercise, exercise_end.

Patient Profile Schema

Patient config is validated with safety ranges.

  • basal_insulin_rate (0.0 - 3.0 U/hr)
  • insulin_sensitivity (10 - 200 mg/dL per unit)
  • carb_factor (3 - 30 g per unit)
  • initial_glucose (40 - 400 mg/dL)
  • insulin_action_duration (60 - 720 min)

Safety Model

  • Hard cutoff for hypoglycemia
  • Rate-of-change protection
  • IOB (insulin-on-board) clamps
  • Time-window caps to prevent stacking
Default critical stop: glucose below 40 mg/dL for 30 minutes triggers termination.

Metrics

Glycemic Control

  • TIR (70-180 mg/dL)
  • TIR Tight (70-140 mg/dL)
  • GMI (Estimated A1C)

Risk Indices

  • LBGI (Low Blood Glucose Index)
  • HBGI (High Blood Glucose Index)
  • CV (Coefficient of Variation)

Troubleshooting

  • Module not found: Ensure you are using a venv (python -m venv).
  • PEP 668 error: Do not install globally on managed Python environments (like Homebrew); use a venv.
  • Early termination: Safety stops simulation if glucose stays below critical threshold (40 mg/dL) for 30 mins.

Ethics and Safety

Critical Warning: This SDK is for research and validation ONLY. It is not a medical device and must not be used for clinical dosing decisions.
